## 四、核心性能瓶颈分析（🔥 高优先级）

### 4.1 全量 DOM 重渲染问题

**位置**：`renderRooms()` 函数（10,387 行）

**问题描述**：
```javascript
function renderRooms(options = {}){
    renderRoomFilters(forceFilters);
    renderRoomGrid(forceGrid);  // 🔴 每次调用都重建整个网格
}
```

**影响**：
- 每次状态变化都触发全量重渲染
- 50 个房间 × 每秒 1 次计时器更新 = 每秒 50 次 DOM 操作
- 在高并发场景下（多人同时登记/退出），会导致：
  - 页面卡顿
  - 输入延迟
  - 动画掉帧

**证据**：
```javascript
// 多处调用 renderRooms() 触发全量重渲染
scheduleRoomsFullRender(reason);  // 3,027 行
renderRooms();  // 5,321 行
renderRooms();  // 6,048 行
```

### 4.2 计时器系统性能问题

**位置**：`updateAllRoomTimers()` 函数（2,992 行）

**问题描述**：
```javascript
window.__globalTimerInterval = setInterval(() => {
    updateAllRoomTimers();  // 🔴 每秒遍历所有房间
}, 1000);
```

**影响**：
- 每秒遍历所有房间（包括空闲房间）
- 每个占用房间都更新 DOM（innerHTML 替换）
- 50 个房间 × 1 秒 = 50 次 DOM 操作/秒

**优化空间**：
- 只更新占用房间（减少 70-80% 计算量）
- 使用 `textContent` 替代 `innerHTML`
- 批量更新 DOM（requestAnimationFrame）

### 4.3 实时同步风暴

**位置**：RealtimeOrchestrator（1,976-2,400 行）

**问题描述**：
- 3 个独立的 Realtime 频道（rooms、students、room_metadata）
- 每个频道的 UPDATE 事件都触发渲染
- 多设备同时操作时，事件风暴导致渲染队列堆积

**证据**：
```javascript
// 每个 UPDATE 事件都触发渲染
.on('postgres_changes', { event: 'UPDATE', ... }, payload => {
    renderRoomCardForced(payload.new.name);  // 🔴 立即渲染
})
```

### 4.4 状态不一致问题

**位置**：乐观更新逻辑（多处）

**问题描述**：
```javascript
// 乐观更新：先修改本地状态
r.occupant_student_name = student;
r.registerTime = Date.now();
r.__tempRegister = true;

// 然后异步写入数据库
await supabaseClient.from('rooms').update(...);
```

**风险**：
- 网络延迟时，本地状态与服务器不一致
- 多设备并发操作时，冲突检测失效
- 版本号机制（version 字段）未严格执行

## 八、立即可执行的快速优化（今天就能做）

### 优化 1：计时器只更新占用房间

**文件**：index.html  
**位置**：`updateAllRoomTimers()` 函数（约 2,992 行）

**修改前**：
```javascript
function updateAllRoomTimers() {
    Store.rooms.forEach(room => {
        // 更新所有房间，包括空闲的
    });
}
```

**修改后**：
```javascript
function updateAllRoomTimers() {
    // 🚀 只更新占用房间
    const occupiedRooms = Store.rooms.filter(r => r.occupant_student_name);

    if (occupiedRooms.length === 0) return;  // 无占用房间时直接返回

    occupiedRooms.forEach(room => {
        const el = document.querySelector(`[data-room='${CSS.escape(room.name)}']`);
        if (!el) return;

        const chip = el.querySelector('.chip');
        if (!chip) return;

        const status = roomStateCalculator.computeStatus(room);

        // 🚀 使用 textContent 替代 innerHTML
        if (chip.textContent !== status.chip) {
            chip.textContent = status.chip;
        }
        if (chip.className !== `chip ${status.class}`) {
            chip.className = `chip ${status.class}`;
        }
    });
}
```

**预期效果**：CPU 占用降低 70%

### 优化 2：实时事件防抖

**位置**：Realtime 频道订阅处理（约 6,200-7,500 行）

**添加防抖工具函数**：
```javascript
// 在 <script> 开头添加
function debounce(func, wait) {
    let timeout;
    return function executedFunction(...args) {
        const later = () => {
            clearTimeout(timeout);
            func(...args);
        };
        clearTimeout(timeout);
        timeout = setTimeout(later, wait);
    };
}

// 创建防抖渲染函数
const debouncedRenderRoom = debounce((roomName) => {
    renderRoomCard(roomName, 'high');
}, 50);
```

**修改 UPDATE 事件处理**：
```javascript
.on('postgres_changes', { event: 'UPDATE', ... }, payload => {
    // 修改前：
    // renderRoomCardForced(payload.new.name);

    // 修改后：
    debouncedRenderRoom(payload.new.name);  // 🚀 防抖
})
```

**预期效果**：重复渲染减少 80%
