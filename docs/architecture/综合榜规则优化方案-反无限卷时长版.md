# 综合榜规则优化方案（反无限卷时长版）

更新时间：2026-05-11（二次更新）
适用范围：综合榜评分规则、相关排行榜展示、AI 分析文案、历史回测与验收脚本
当前状态：方案设计稿，已补充第二轮“收敛参数”模拟与音符币保护边界，未实施数据库与前端代码改动

## 1. 背景与目标

当前综合榜已经不再是纯总时长榜，但仍存在两个容易被误解或被利用的问题：

1. 优秀学生在 AI 解读和部分评分语义上，容易被引导成“要想继续靠前，就得每周继续增加练琴时长”。
2. 如果未来只强调“稳定达标”，又可能出现“本来练得不多，但因为个人目标低且周周达标，排名反而过高”的反向失衡。

本方案的目标是同时解决这两类问题：

- 不鼓励学生通过无上限增加练琴时长来保住排名。
- 不允许低目标学生仅靠“稳定打卡”轻易压过绝对水平更高的学生。
- 保留对中段学生的正向激励，让“从未达标走向稳定达标”仍然是有效成长路径。
- 尽量复用现有五维结构 `W / B / T / M / A`，降低系统重构成本。

一句话目标：

综合榜不再奖励“谁一直越练越多”，而是奖励“谁稳定达标、绝对水平够高、最近没有明显回落或异常”。

## 2. 新规则总原则

新的综合榜建议围绕三层能力展开：

1. 达标能力：是否能稳定完成自己的练琴目标。
2. 水平能力：长期绝对练琴水平是否足够高。
3. 稳定能力：最近几周是否保持状态、避免回落、减少异常。

新的导向不是取消总量，而是降低“持续堆总量”的主导性：

- 达标率决定你有没有资格拿高分。
- 绝对水平决定你能不能冲到榜首。
- 稳定性决定你能不能长期待在前列。

## 3. 建议的新五维定义

### 3.1 W：本周进度分

建议从“本周工作日累计进度相对个人目标的实时完成度”升级为“双锚进度分”：

- 个人目标锚：`personal_target_week = clamp(personal_mean_daily * 5, 240, 720)`
- 专业地板锚：`major_floor_week = 专业参考周目标 * 0.75`
- 最终目标：`final_target_week = max(personal_target_week, major_floor_week)`

再基于两个比率计算：

- `completion_ratio = 本周有效分钟 / final_target_week`
- `absolute_ratio = 本周有效分钟 / 专业参考周目标`

建议效果：

- `0% ~ 100%` 正常拉升。
- `100% ~ 115%` 仅给极小额外奖励。
- `>115%` 基本封顶，不再鼓励无限加量。

目标：

- 防止学生单靠“无限往上堆本周分钟数”获得持续收益。
- 防止历史基础很低的学生因为目标过低而轻松拿满分。

### 3.2 B：短期进步分

当前语义是“最近 1 个活跃周 vs 前 1 个活跃周”，容易演化成“下周必须比本周练更多”。

建议改为比较“目标完成率的变化”，而不是裸分钟差：

- `week_n_completion = min(week_n_valid_minutes / week_n_target, 1.15)`
- `B = 最近1周完成率 vs 前1周完成率`

建议加入平台保护：

- 差值在 `±5%` 内视为持平。
- 如果两周都 `>=95%` 达标且波动不大，不应判为“短期进步差”。

目标：

- 已经稳定达标的学生不需要每周继续创新高。
- 从未达标到接近达标的学生，仍然能清楚获得进步奖励。

### 3.3 T：近期趋势分

当前语义是“最近 2 个活跃周均值 vs 前 2 个活跃周均值”，容易被误读为“必须一直往上加量”。

建议改为比较“最近两段的平均完成率”，并引入平台保护：

- `T = 最近2周平均完成率 vs 前2周平均完成率`
- 如果两段都已经落在 `95% ~ 105%` 的稳定达标区间，默认给中高分，不因“没有继续增加”而扣分。

重点只对以下情况明显扣分：

- 从达标区掉回未达标区。
- 最近两周出现明显断档。
- 波动突然增大或异常显著上升。

目标：

- 把“趋势”从“持续增量”改为“持续稳定”。

### 3.4 M：稳定达标分

这一项建议升级为核心维度，明确奖励“连续稳定完成目标”。

建议定义：

- 最近 4 个活跃周的加权达标率。
- 最近周权重更高，例如 `0.4 / 0.3 / 0.2 / 0.1`。
- 同时引入波动惩罚和断档惩罚。

目标：

- 单周爆量不再能替代连续稳定。
- “稳定好学生”会更合理地向前移动。

### 3.5 A：长期积累分

这一项是防止“低目标学生稳定霸榜”的关键。

建议从“长期总量倾向”改为“两部分组合”：

1. 同专业长期绝对水平分位。
2. 长期活跃稳定度。

建议结构：

- `A = 0.7 * 同专业长期水平分位 + 0.3 * 长期稳定活跃度`

其中长期水平分位不建议直接用生涯总分钟，而更建议用：

- 最近 8~12 个活跃周的周有效时长中位数
- 或最近 8~12 个活跃周的稳健平均值

目标：

- 练得少但稳定的人，`M` 可以不低，但 `A` 不会虚高。
- 练得多但很乱的人，`A` 可能高，但 `M` 会拉住。
- 真正高水平又稳定的人，`A` 和 `M` 都强，综合排名才更稳。

## 4. 建议的新权重

建议成熟期学生调整为：

- `W 20%`
- `B 15%`
- `T 15%`
- `M 25%`
- `A 25%`

设计意图：

- `W` 保留重要性，但不再主导。
- `B/T` 保留成长感，但不再强推“周周加量”。
- `M/A` 构成新的主骨架，分别代表“稳定”和“水平”。

过渡期学生可建议为：

- `W 25%`
- `B 15%`
- `T 15%`
- `M 20%`
- `A 25%`

新生期学生可建议为：

- `W 40%`
- `B 10%`
- `T 5%`
- `M 15%`
- `A 30%`

注意：

- 新生期的 `A` 需要强收缩，不能直接把小样本总量放大。
- 若实施风险控制优先，可先只调整成熟期与过渡期。

## 5. 建议的综合榜基础门槛

为防止低质量样本或极薄样本扰动综合榜，建议增加基础门槛：

- 本周至少 `2` 次有效练琴，或
- 本周至少 `120` 分钟有效时长
- 本周异常率不高于 `40%`
- 最近 4 个活跃周至少有 `2` 周为有效活跃周

冷启动学生仍允许上榜，但建议：

- `B/T/M/A` 收缩到中性附近
- 让 `W` 承担主要短期解释权

## 6. 对四类学生的预期影响

### 6.1 练得少但稳定的人

- `W/M` 可能不低
- `A` 不会太高
- 综合分应在中上区间，但不应轻易霸榜

### 6.2 练得多但不稳定的人

- `A` 偏高
- `M/B/T` 被拖低
- 综合分不会因为单周高总量而虚高

### 6.3 高水平且稳定的人

- `W/M/A` 都较强
- 综合榜前列更合理

### 6.4 单周爆量冲榜的人

- `W` 可获有限收益
- `M/T` 不会同步走高
- 若异常多，还会被纪律惩罚压住

## 8. 是否涉及较大改动

结论：是，中等偏大的规则改动，但不是“架构级破坏性重构”。

更准确地说：

- 不是数据库表结构大重构。
- 不是 API 形状大变更。
- 不是前端模块拆迁级别改造。
- 但它是“评分核心语义重算级别”的重要规则变更。

可分为三个层面的影响：

### 8.1 小改动层

- AI 文案解释
- 排行榜说明文案
- README / 文档更新

这类改动风险低。

### 8.2 中改动层

- W 公式重写为双锚模型
- B/T 从比裸分钟改为比完成率
- 调试与验证脚本同步更新

这类改动风险中等，需要回测。

### 8.3 大改动层

- A 的长期积累逻辑重定义
- M/A 权重重配
- 历史快照与周榜回放一致性验证
- 音符币奖励与历史榜单比较影响评估

这类改动虽然不一定要改表结构，但对榜单排序结果影响较大，属于“业务结果大变化”级别。

## 9. 是否会产生破坏

结论：会产生“结果层破坏”，但不一定产生“系统层破坏”。

### 9.1 不太会破坏的部分

- 现有数据库主要表结构可大概率复用。
- 现有前端页面结构可大概率复用。
- 现有函数调用入口、定时任务、AI 分析链路可继续复用。

### 9.2 会发生明显变化的部分

1. 综合榜名次会变化
   - 这是预期内变化。
   - 一些“单周猛冲型”学生可能下滑。
   - 一些“高水平且稳定型”学生可能上升。

2. 历史周榜与当前规则的可比性会变差
   - 如果不做版本标记，后续解释“为什么这个月前后榜单风格不同”会比较麻烦。

3. 自动音符币奖励可能受影响
   - 综合榜前三、前十等发币逻辑若依赖名次，奖励对象可能改变。

4. AI 解读必须同步
   - 否则会出现“分数逻辑换了，但 AI 还按旧逻辑解释”的错位。

## 10. 风险等级判断

建议按下列风险理解：

- 技术实现风险：中
- 数据结构破坏风险：低
- 排名结果变化风险：高
- 奖励结果变化风险：高
- 用户感知变化风险：高

因此，这不是“不能做”的改动，而是“必须先回测、再灰度”的改动。

## 11. 推荐实施顺序

### 阶段 1：低风险试点

1. 先只改 AI 文案和规则说明文案
2. 明确告诉用户：达标后重点看稳定性，不是无限加量

状态：

- 此阶段已完成第一步 AI 文案修正

### 阶段 2：中风险规则试点

3. 只改 `B/T` 为“完成率比较”
4. 保持当前 `A/M/W` 不动
5. 回测最近 8~12 周榜单差异

目的：

- 先验证能否明显缓解“周周加量”导向
- 同时观察是否出现异常的低目标学生上浮

### 阶段 3：完整规则升级

6. 改 `W` 为双锚模型
7. 提升 `M/A` 权重
8. 重定义 `A` 为长期绝对水平 + 长期稳定活跃度
9. 做历史周榜、自动发币、AI 解读三条链路联调

## 12. 最终判断

这套规则优化是值得做的，但不能直接一次性“全量切换”。

最稳的策略是：

- 先完成文案和解释层修正
- 再做 `B/T` 的中等强度调整
- 最后再做 `W/A/M` 的完整重构

如果直接一步切到底层评分，会导致：

- 排名风格突然变化
- 奖励结果变化较大
- 历史榜单解释成本上升

如果按阶段推进，则可以：

- 控制风险
- 保留可回退性
- 更容易向老师和学生解释“榜单为什么变得更合理”

## 13. 当前结论摘要

是否值得改：值得
是否属于较大改动：是，业务规则中到大改
是否会破坏数据库结构：大概率不会
是否会破坏榜单结果稳定性：会，且是预期内变化
是否建议直接一次性实施：不建议
是否建议分阶段灰度：建议

## 14. 第二轮“收敛参数”模拟更新

本轮更新不是直接上线的新规则，而是对第一轮方案进一步做“保守收敛”后的模拟稿，目标是两点：

1. 继续压制“无限卷时长”导向。
2. 同时避免榜单风格一次性变化过猛，降低对周榜、老师解释口径和学生预期的冲击。

第二轮的思路不是推翻第一轮，而是在第一轮原则上把参数收得更稳：

- 降低单次大改的幅度，优先减少翻榜震荡。
- 保留“达标后不必周周创新高”的导向。
- 强化长期绝对水平，但不让它重新退化成总时长霸榜。
- 明确把“历史音符币不追溯、不改写”作为实施前提。

### 14.1 第二轮模拟参数包

建议把第二轮模拟参数收敛为以下保守版本。

#### A. W 维度：本周进度分

继续采用双锚目标，但收紧封顶与过量奖励：

- `personal_target_week = clamp(personal_mean_daily * 5, 270, 1500)`
- `major_floor_week = 专业参考周目标 * 0.80`
- `final_target_week = max(personal_target_week, major_floor_week)`

评分导向建议：

- `completion_ratio <= 1.00`：正常拉升
- `1.00 ~ 1.10`：仅保留少量奖励
- `>1.10`：基本封顶

相较第一轮，第二轮把可获益的超额区从 `115%` 再压到 `110%`，进一步减少“靠多练一点点就持续拉开差距”的动力。

#### B. B 维度：短期进步分

继续改为“完成率差”而不是“裸分钟差”，但把平台保护做得更宽一些：

- 如果两周完成率差在 `±6%` 以内，视为基本持平
- 如果两周都 `>= 92%`，即使第二周没有继续上涨，也不判为负进步
- 只有当最近一周明显跌出达标区，才给出清晰扣分

这能减少优秀学生在高位小波动时被不必要地下压。

#### C. T 维度：近期趋势分

继续比较最近两段平均完成率，但把稳定达标平台区放宽为：

- `92% ~ 108%` 视为稳定达标区
- 两段都处于该区间时，给中高分，不要求继续上升
- 只有“从达标区掉到明显未达标”或“连续两周显著回落”时，才明显扣分

这一版比第一轮更保守，主要是减少“明明表现稳定，却因为没继续增加时长而掉分”的体感。

#### D. M 维度：稳定达标分

第二轮保留“最近 4 个活跃周加权达标”思路，但权重收敛为：

- `0.35 / 0.30 / 0.20 / 0.15`

比第一轮的 `0.4 / 0.3 / 0.2 / 0.1` 更平滑，避免单周异常对总结果的放大过强。

#### E. A 维度：长期积累分

第二轮建议继续用“两部分组合”，但把长期绝对水平分量进一步做稳健化：

- `A = 0.75 * 同专业长期水平分位 + 0.25 * 长期稳定活跃度`

其中“长期水平分位”建议基于：

- 最近 `8~12` 个活跃周的周有效时长中位数
- 必要时可增加温和截尾，避免个别极端高周把长期水平拉得过高

这样既不会让“练得少但稳定”的学生轻易冲到最前，也不会让“偶尔超长爆量”的学生占到不合理优势。

### 14.2 第二轮建议权重

第二轮建议采用比第一轮更保守的成熟期权重：

- `W 22%`
- `B 12%`
- `T 12%`
- `M 24%`
- `A 30%`

设计意图：

- 继续降低 `W/B/T` 对“短期冲量”的敏感度
- 让 `A/M` 成为更稳定的主骨架
- 但不把 `W` 压得太低，避免学生端觉得“这周努力完全没意义”

过渡期学生可建议为：

- `W 28%`
- `B 12%`
- `T 10%`
- `M 20%`
- `A 30%`

新生期学生可建议为：

- `W 42%`
- `B 8%`
- `T 5%`
- `M 15%`
- `A 30%`

说明：

- 这些是第二轮模拟参数，不是已部署权重。
- 第二轮比第一轮更强调“高位稳定”和“长期绝对水平”，但收敛得更慢，适合先做历史回测与模拟对照。

## 15. 音符币保护边界：本次更新必须满足的硬约束

这次文档更新和后续第二轮模拟，必须建立在“不影响任何历史音符币统计与发放”的前提上。这个前提不是建议，而是实施红线。

### 15.1 明确不允许触碰的对象

以下对象在第二轮模拟与未来上线中，都不得为了新规则去回写、删改或重算历史结果：

1. `public.student_coins`
2. `public.coin_transactions`
3. `public.weekly_coin_reward_log`
4. `public.weekly_coin_reward_detail`
5. 已写入的 `public.weekly_leaderboard_history` 历史周快照

原因：

- `reward_weekly_coins()` 已经按当时榜单结果发过币。
- `weekly_coin_reward_log` / `weekly_coin_reward_detail` 是审计凭证。
- `coin_transactions` 是已经发生过的真实流水。

这些都属于“历史事实”，不能因为新评分方案更合理，就回头改写。

### 15.2 第二轮模拟的正确边界

第二轮模拟只允许做以下事情：

1. 新建文档说明
2. 新建或修改回测 SQL / debug SQL / verify SQL
3. 用只读方式重放历史周数据，生成“如果按新规则会怎样”的模拟结果
4. 生成单独的模拟报告、对比表、差异分析

第二轮模拟不允许做以下事情：

1. 不允许修改 `reward_weekly_coins()` 当前线上逻辑
2. 不允许重跑历史发币
3. 不允许删除或重写历史 `weekly_coin_reward_log`
4. 不允许删除或重写历史 `weekly_coin_reward_detail`
5. 不允许把模拟榜单结果覆盖进历史 `weekly_leaderboard_history`

### 15.3 如果未来正式实施新规则，必须采用“生效日切换”，不能历史回灌

如果第二轮参数最终决定上线，建议的安全实施方式是：

1. 保留历史周榜、历史音符币、历史流水完全不动
2. 从一个明确的未来周一开始启用新规则
3. 从该生效周开始，新的 `get_weekly_leaderboards()` 结果按新规则产生
4. 从该生效周开始，新的 `reward_weekly_coins()` 依据新榜单发放
5. 历史周仍然按旧规则解释，必要时在文档中注明“规则版本切换日”

这样可以同时保证：

- 审计连续性
- 音符币账务连续性
- 老周榜解释稳定性
- 新规则的可控上线

### 15.4 需要额外增加的实施保护

为避免后续误操作，正式实施前建议补充以下保护：

1. 在文档中明确“新规则只影响生效周之后的榜单与发币”
2. 为回测脚本和模拟脚本统一加上“只读 / 不落库”说明
3. 如后续要做版本治理，建议为周榜备份增加“规则版本”标记，但仅用于未来周，不倒灌历史周
4. 严禁在新规则上线后为了“对齐历史”去批量清理或重发旧音符币流水

## 16. 第二轮更新后的结论

第二轮“收敛参数”模拟可以继续推进，但边界必须收紧成下面这句话：

新规则可以继续模拟、回测、对比，甚至未来可以从某个新周起正式启用；但任何历史音符币统计、历史发放记录、历史榜单快照，都不允许因为这轮优化而被回写或重算。

因此，当前建议的执行顺序调整为：

1. 先完成本次更新文档
2. 再补“第二轮收敛参数”只读回测脚本与差异报告
3. 若模拟结果满意，再设计“未来生效周切换”方案
4. 在未明确生效周之前，不改线上发币逻辑，不改历史发币结果
