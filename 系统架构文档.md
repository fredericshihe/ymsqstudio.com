## 整体架构概览

```
┌──────────────────────────────────────────────────────────────┐
│                        前端（浏览器）                          │
│                                                              │
│  练琴跟踪优化简化版本.html   dashboard.html   admin_coins.html │
│        学生端首页 & 详情        管理员大盘         音符币后台    │
└─────────────────┬────────────────┬─────────────────┬─────────┘
                  │ supabase-js    │ fetch REST API   │ supabase-js
                  ▼                ▼                  ▼
┌──────────────────────────────────────────────────────────────┐
│                  Supabase（PostgreSQL + Realtime）             │
│                                                              │
│  RPC 函数                  数据表                  定时任务    │
│  get_weekly_leaderboards   practice_sessions       pg_cron    │
│  adjust_student_coins      student_score_history              │
│  backup_weekly_leaderboards student_baseline                  │
│                            student_coins                      │
│                            coin_transactions                  │
│                            weekly_leaderboard_history         │
└──────────────────────────────────────────────────────────────┘
```

**核心技术栈：**
- 前端：纯 HTML + CSS + JavaScript（无框架），通过 `@supabase/supabase-js@2` 连接数据库
- 后端：Supabase（托管 PostgreSQL），使用 RPC 函数、数据库触发器、pg_cron 定时任务
- 实时推送：Supabase Realtime（WebSocket），监听 `practice_logs` 表变化

## 排行榜系统

### 排行榜涉及数据库表

#### 1. `practice_sessions`（练琴会话记录）
排行榜计算的**原始数据来源**，记录每次完整的练琴会话。

| 字段 | 类型 | 说明 |
|------|------|------|
| `student_name` | TEXT | 学生姓名 |
| `session_start` | TIMESTAMPTZ | 练琴开始时间 |
| `session_end` | TIMESTAMPTZ | 练琴结束时间 |
| `cleaned_duration` | NUMERIC | 有效练琴时长（分钟，去除异常） |
| `is_outlier` | BOOLEAN | 是否为异常练琴记录 |
| `outlier_reason` | TEXT | 异常原因（too_long / meal_break 等） |
| `room_name` | TEXT | 练琴室名称 |

#### 2. `student_baseline`（学生基线数据）
每位学生的**最新画像数据**，由后端触发器 `compute_student_score` 自动更新。

| 字段 | 类型 | 说明 |
|------|------|------|
| `student_name` | TEXT PK | 学生姓名 |
| `student_major` | TEXT | 专业 |
| `student_grade` | TEXT | 年级 |
| `composite_score` | NUMERIC | 综合排名分（0-100） |
| `alpha` | NUMERIC | 基线可信度 α（0.5-1.0，越高越稳定） |
| `trend_score` | NUMERIC | 趋势分（0-1，>0.5 说明近期上升） |
| `mean_duration` | NUMERIC | 历史均练琴时长（分钟） |
| `record_count` | INTEGER | 历史有效记录总数 |
| `outlier_rate` | NUMERIC | 异常率 |
| `last_updated` | TIMESTAMPTZ | 最后更新时间 |

#### 3. `student_score_history`（成长分历史快照）
每周一自动写入的**历史快照**，记录每位学生每周的各项维度分数。

| 字段 | 类型 | 说明 |
|------|------|------|
| `student_name` | TEXT | 学生姓名 |
| `snapshot_date` | DATE | 快照日期（每周一） |
| `composite_score` | NUMERIC | 当周综合排名分 |
| `raw_score` | NUMERIC | 个人原始成长值（0-1） |
| `trend_score` | NUMERIC | 趋势分 |
| `baseline_score` | NUMERIC | 与上周对比的进步分 |
| `mean_duration` | NUMERIC | 均练时长 |
| `record_count` | INTEGER | 累计记录数 |
| `outlier_rate` | NUMERIC | 异常率 |

#### 4. `weekly_leaderboard_history`（排行榜历史备份）
每周五 21:30 自动备份的**历史榜单归档**。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID PK | 主键 |
| `backup_date` | DATE | 备份执行日期 |
| `week_monday` | DATE | 归属的周一（用于按周查询） |
| `board` | TEXT | 榜单名（综合榜/进步榜/稳定榜/守则榜） |
| `rank_no` | INTEGER | 名次 |
| `student_name` | TEXT | 学生姓名 |
| `student_major` | TEXT | 专业 |
| `student_grade` | TEXT | 年级 |
| `display_score` | NUMERIC | 综合分 |
| `alpha` | NUMERIC | 可信度 α |
| `trend_score` | NUMERIC | 趋势分 |
| `mean_duration` | NUMERIC | 均练时长 |
| `record_count` | INTEGER | 记录数 |
| `recent10_outlier_rate` | NUMERIC | 近10条异常率 |
| `recent10_mean_dur` | NUMERIC | 近10条均练时长 |
| `recent10_count` | INTEGER | 近10条记录数 |
| `created_at` | TIMESTAMPTZ | 写入时间 |

**索引：**
- `idx_wlh_week_board`：`(week_monday, board)` — 按周 + 榜单快速查询
- `idx_wlh_student`：`(student_name)` — 按学生查历史

### 排行榜涉及数据库函数

#### `public.get_weekly_leaderboards()`

**作用：** 一次 RPC 调用返回本周全部四个榜单的完整数据。

**文件：** `leaderboard_rpc.sql`

**返回类型：** 表，字段如下：

| 返回字段 | 类型 | 说明 |
|----------|------|------|
| `board` | TEXT | 榜单名：综合榜 / 进步榜 / 稳定榜 / 守则榜 |
| `rank_no` | INTEGER | 名次（RANK 函数，允许并列） |
| `student_name` | TEXT | 学生姓名 |
| `student_major` | TEXT | 专业 |
| `student_grade` | TEXT | 年级 |
| `display_score` | NUMERIC | 综合分（保留1位小数展示） |
| `alpha` | NUMERIC | 可信度 α |
| `trend_score` | NUMERIC | **进步榜**：本周综合分 − 上周综合分的 delta 整数；**其余三榜**：真实趋势分（0~1）|
| `mean_duration` | NUMERIC | 均练时长（分钟） |
| `record_count` | INTEGER | 历史总记录数 |
| `recent10_outlier_rate` | NUMERIC | 近10条异常率 |
| `recent10_mean_dur` | NUMERIC | 近10条均练时长 |
| `recent10_count` | INTEGER | 近10条记录数（12周内） |

**内部 CTE 逻辑链：**

```
practice_sessions ──► recent10（近12周最多10条）
                           │
student_baseline ──────────┤
student_score_history ─────┤
                           ▼
practice_sessions ──► week_cnt（本周次数）
                           │
student_score_history ──► week_scores（本周快照）
                           │
student_score_history ──► last_week_scores（上周快照，进步榜基准）
                           │
                           ▼
                      ranked_pool（候选池：本周有练+分>0）
                           │
          ┌────────────────┼────────────────┬──────────────┐
          ▼                ▼                ▼              ▼
         comp            prog            stable          rules
       综合榜           进步榜           稳定榜          守则榜
       （全部）    （JOIN上周快照，       （α最高）      （异常率最低）
                   按delta排序）
          │
          └──────── UNION ALL ────────► 最终结果集
                                        ORDER BY board, rank_no
```

**权限：** `GRANT EXECUTE TO anon, authenticated`
**属性：** `STABLE`（同一事务内相同输入返回相同结果，可缓存）

---

#### `public.backup_weekly_leaderboards()`

**作用：** 将当前 `get_weekly_leaderboards()` 的结果完整存入 `weekly_leaderboard_history` 表。同一周内重复调用会先删旧数据再写入，确保每周只有一份最新备份。

**文件：** `leaderboard_backup.sql`

**触发方式：** `pg_cron` 定时任务自动调用（每周五 UTC 13:30 = 北京时间 21:30）

**权限：** `SECURITY DEFINER`（以函数创建者的最高权限执行，无需单独授权）

### 四大榜单规则详解

| 榜单 | 核心排序字段 | 次级排序（并列时） | 上榜条件 |
|------|------------|----------------|---------|
| **综合榜** | `display_score` ↓ | `mean_duration` ↓ → `record_count` ↓ | 本周有练琴 + 综合分 > 0 |
| **进步榜** | `(display_score - 上周composite_score)` ↓ | `display_score` ↓ → `mean_duration` ↓ | 上周有练琴快照 + 本周综合分 > 上周 + α ≥ 0.50 + 近10条异常率 ≤ 70% + 综合分 ≥ 15 |
| **稳定榜** | `alpha` ↓ | `mean_duration` ↓ | α ≥ 0.65 + 近10条记录 ≥ 10条 + 近10条异常率 ≤ 35% |
| **守则榜** | `recent10_outlier_rate` ↑（越低越好） | `record_count` ↓ → `mean_dur` ↓ | 近10条 ≥ 5条 + 均时 > 30min + 异常率 ≤ 50% + α ≥ 0.60 + **本周练琴 ≥ 5次** |

**关键概念：**
- **α（基线可信度）**：`有效记录数 / (有效记录数 + 8)`，记录越多越稳定，最大趋近 1.0
- **异常练琴**：单次练琴 `is_outlier = true`，包括 `too_long`（时长异常超长）和 `meal_break`（含饭点时段）
- `RANK()` 函数：完全相同的多维排序值才会真正并列，加入多级排序后并列概率极低
- **进步榜 `trend_score` 字段复用**：进步榜行的 `trend_score` 列存储的是 `本周综合分 - 上周综合分` 的 delta 整数（例如 +24），而非原始趋势分（0~1）。其余三个榜单的 `trend_score` 仍为真实趋势分。

## 音符币系统

### 音符币涉及数据库表

#### 1. `student_coins`（学生余额表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `student_name` | TEXT PK | 学生姓名（主键） |
| `balance` | INTEGER | 当前音符币余额，默认 0 |
| `updated_at` | TIMESTAMPTZ | 最后更新时间 |

**特点：**
- 每个学生只有一行记录
- `balance` 永远是最新余额，无需从流水加总计算

#### 2. `coin_transactions`（音符币流水记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID PK | 主键（自动生成） |
| `student_name` | TEXT | 学生姓名 |
| `amount` | INTEGER | 变动数量（**正数=增加，负数=扣除**） |
| `balance_after` | INTEGER | 变动后的最新余额（冗余存储，方便展示无需回溯计算） |
| `reason` | TEXT | 备注说明（必填，如"综合榜第一名奖励"） |
| `transaction_type` | TEXT | `'manual'`=管理员手动操作，`'auto_reward'`=系统自动结算 |
| `created_at` | TIMESTAMPTZ | 记录创建时间 |

**索引：**
- `idx_coin_tx_student`：`(student_name, created_at DESC)` — 按学生查流水，时间倒序

**RLS（行级安全策略）：**
- 读取：允许所有人（`anon` + `authenticated`）查询
- 写入：仅通过 `adjust_student_coins` RPC 函数写入（SECURITY DEFINER 权限）

---

### 音符币涉及数据库函数与视图

#### `public.adjust_student_coins(p_student_name, p_amount, p_reason, p_type)`

**作用：** 原子操作，确保"更新余额"和"写入流水"同时成功，不会出现数据不一致。

**文件：** `setup_coins_db.sql`

**参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `p_student_name` | TEXT | 学生姓名 |
| `p_amount` | INTEGER | 变动数量（正=加，负=减） |
| `p_reason` | TEXT | 备注说明 |
| `p_type` | TEXT | 操作类型，默认 `'manual'`；自动结算传 `'auto_reward'` |

**返回：** `INTEGER`（变动后的最新余额）

**内部执行步骤：**
1. `INSERT ... ON CONFLICT DO NOTHING`：若学生不在余额表，先初始化为 0
2. `SELECT ... FOR UPDATE`：行锁，防止并发修改
3. 计算新余额：`v_new_balance := v_current_balance + p_amount`
4. `UPDATE student_coins`：更新余额
5. `INSERT coin_transactions`：写入流水记录
6. 返回新余额

**权限：** `SECURITY DEFINER` + `GRANT EXECUTE TO anon, authenticated`

---

#### `public.vw_student_coin_balances`（视图）

**作用：** 将 `student_database`（学生基础信息）与 `student_coins`（余额）联合查询，方便管理后台一次性加载学生列表。

```sql
SELECT 
    sd.name AS student_name,
    sd.major AS student_major,
    sd.grade AS student_grade,
    COALESCE(sc.balance, 0) AS balance,  -- 没有记录的学生默认显示 0
    sc.updated_at
FROM student_database sd
LEFT JOIN student_coins sc ON sd.name = sc.student_name;
```

**权限：** `GRANT SELECT TO anon, authenticated`
> Supabase 后台会显示 `UNRESTRICTED` 红色标签，属于正常现象，不影响功能。

## 定时任务

| 任务名称 | Cron 表达式（UTC） | 北京时间 | 执行函数 | 说明 |
|---------|-----------------|---------|---------|------|
| `backup_weekly_leaderboards_job` | `30 13 * * 5` | 每周五 21:30 | `backup_weekly_leaderboards()` | 将当周四榜快照写入 `weekly_leaderboard_history` |

**管理命令：**
```sql
-- 查看所有定时任务
SELECT * FROM cron.job;

-- 手动触发备份（测试用）
SELECT public.backup_weekly_leaderboards();

-- 取消任务
```
