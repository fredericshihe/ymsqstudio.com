**核心技术栈：**
- 前端：纯 HTML + CSS + JavaScript（无框架），通过 `@supabase/supabase-js@2` 连接数据库

- 实时推送：Supabase Realtime（WebSocket），监听 `practice_logs` 表变化

## 排行榜系统

### 排行榜涉及数据库表

#### 1. `practice_sessions`（练琴会话记录）
排行榜计算的**原始数据来源**，记录每次完整的练琴会话。

| 字段 | 类型 | 说明 |
|------|------|------|
| `student_name` | TEXT | 学生姓名 |
| `session_start` | TIMESTAMPTZ | 练琴开始时间 |
| `session_end` | TIMESTAMPTZ | 练琴结束时间 |
| `cleaned_duration` | NUMERIC | 有效练琴时长（分钟，去除异常） |
| `is_outlier` | BOOLEAN | 是否为异常练琴记录 |
| `outlier_reason` | TEXT | 异常原因（too_long / meal_break 等） |
| `room_name` | TEXT | 练琴室名称 |

#### 2. `student_baseline`（学生基线数据）
每位学生的**最新画像数据**，由后端触发器 `compute_student_score` 自动更新。

| 字段 | 类型 | 说明 |
|------|------|------|
| `student_name` | TEXT PK | 学生姓名 |
| `student_major` | TEXT | 专业 |
| `student_grade` | TEXT | 年级 |
| `composite_score` | NUMERIC | 综合排名分（0-100） |
| `alpha` | NUMERIC | 基线可信度 α（0.5-1.0，越高越稳定） |
| `trend_score` | NUMERIC | 趋势分（0-1，>0.5 说明近期上升） |
| `mean_duration` | NUMERIC | 历史均练琴时长（分钟） |
| `record_count` | INTEGER | 历史有效记录总数 |
| `outlier_rate` | NUMERIC | 异常率 |
| `last_updated` | TIMESTAMPTZ | 最后更新时间 |

#### 3. `student_score_history`（成长分历史快照）
每周一自动写入的**历史快照**，记录每位学生每周的各项维度分数。

| 字段 | 类型 | 说明 |
|------|------|------|

| `snapshot_date` | DATE | 快照日期（每周一） |
| `composite_score` | NUMERIC | 当周综合排名分 |
| `raw_score` | NUMERIC | 个人原始成长值（0-1） |
| `trend_score` | NUMERIC | 趋势分 |
| `baseline_score` | NUMERIC | 与上周对比的进步分 |
| `mean_duration` | NUMERIC | 均练时长 |
| `record_count` | INTEGER | 累计记录数 |
| `outlier_rate` | NUMERIC | 异常率 |

#### 4. `weekly_leaderboard_history`（排行榜历史备份）
每周五 21:30 自动备份的**历史榜单归档**。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID PK | 主键 |
| `backup_date` | DATE | 备份执行日期 |
| `week_monday` | DATE | 归属的周一（用于按周查询） |
| `board` | TEXT | 榜单名（综合榜/进步榜/稳定榜/守则榜） |
| `rank_no` | INTEGER | 名次 |
| `student_name` | TEXT | 学生姓名 |
| `student_major` | TEXT | 专业 |
| `student_grade` | TEXT | 年级 |
| `display_score` | NUMERIC | 综合分 |
| `alpha` | NUMERIC | 可信度 α |
| `trend_score` | NUMERIC | 趋势分 |
| `mean_duration` | NUMERIC | 均练时长 |
| `record_count` | INTEGER | 记录数 |
| `recent10_outlier_rate` | NUMERIC | 近10条异常率 |
| `recent10_mean_dur` | NUMERIC | 近10条均练时长 |
| `recent10_count` | INTEGER | 近10条记录数 |

**索引：**

### 排行榜涉及数据库函数

#### `public.get_weekly_leaderboards()`

**作用：** 一次 RPC 调用返回本周全部四个榜单的完整数据。

**文件：** `leaderboard_rpc.sql`

**返回类型：** 表，字段如下：

| 返回字段 | 类型 | 说明 |
|----------|------|------|
| `board` | TEXT | 榜单名：综合榜 / 进步榜 / 稳定榜 / 守则榜 |
| `rank_no` | INTEGER | 名次（RANK 函数，允许并列） |
| `student_name` | TEXT | 学生姓名 |
| `student_major` | TEXT | 专业 |
| `student_grade` | TEXT | 年级 |
| `display_score` | NUMERIC | 综合分（保留1位小数展示） |
| `alpha` | NUMERIC | 可信度 α |
| `trend_score` | NUMERIC | **进步榜**：本周综合分 − 上周综合分的 delta 整数；**其余三榜**：真实趋势分（0~1）|
| `mean_duration` | NUMERIC | 均练时长（分钟） |
| `record_count` | INTEGER | 历史总记录数 |
| `recent10_outlier_rate` | NUMERIC | 近10条异常率 |
| `recent10_mean_dur` | NUMERIC | 近10条均练时长 |
| `recent10_count` | INTEGER | 近10条记录数（12周内） |

**内部 CTE 逻辑链：**

```
practice_sessions ──► recent10（近12周最多10条）
                           │
student_baseline ──────────┤
student_score_history ─────┤
                           ▼
practice_sessions ──► week_cnt（本周次数）
                           │
student_score_history ──► week_scores（本周快照）
                           │
student_score_history ──► last_week_scores（最近活跃周快照，最多回溯12周，进步榜基准）
                           │
                           ▼
                      ranked_pool（候选池：本周有练+分>0）
                           │
                           ▼
                    comp（综合榜，全部入池学生）
                           │
                    comp_top10（综合榜 Top 10 名单）
                           │  ← 专项榜过滤：排除 comp_top10
          ┌────────────────┼────────────────┐
          ▼                ▼                ▼
         prog            stable           rules
       进步榜            稳定榜           守则榜
  （绝对涨分↓，       （α↓ 一致性，      （异常率↑最低，
   本周≥2 + 涨>0）     cnt≥8 等）        本周≥3 等）
          │
          └──────── UNION ALL ────────► 最终结果集
                                        ORDER BY board, rank_no
```

**权限：** `GRANT EXECUTE TO anon, authenticated`
**属性：** `STABLE`（同一事务内相同输入返回相同结果，可缓存）

#### `public.backup_weekly_leaderboards()`

**作用：** 将当前 `get_weekly_leaderboards()` 的结果完整存入 `weekly_leaderboard_history` 表。同一周内重复调用会先删旧数据再写入，确保每周只有一份最新备份。

**文件：** `leaderboard_backup.sql`

**触发方式：** `pg_cron` 定时任务自动调用（每周五 UTC 13:30 = 北京时间 21:30）

**权限：** `SECURITY DEFINER`（以函数创建者的最高权限执行，无需单独授权）

### 四大榜单规则详解

| 榜单 | 核心排序字段 | 次级排序（并列时） | 上榜条件 |
|------|------------|----------------|---------|
| **综合榜** | `display_score` ↓ | `mean_duration` ↓ → `record_count` ↓ | 本周有练琴 + 综合分 > 0 |
| **进步榜** | `display_score - lw_composite`（**绝对涨分**）↓ | `display_score` ↓ → `mean_duration` ↓ | 有历史周快照 + 绝对涨分 > 0 + 本周练琴 ≥ 2次 + 近10条异常率 ≤ 50% + **综合榜 Top 10 自动退出** |
| **稳定榜** | `alpha`（一致性）↓ | `recent10_mean_dur` ↓ → `outlier_rate` ↑ | α ≥ 0.55 + 近10条 ≥ 8条 + 近10条异常率 ≤ 40% + **综合榜 Top 10 自动退出** |
| **守则榜** | `recent10_outlier_rate` ↑（越低越好） | `week_sessions`（本周次数）↓ → `mean_dur` ↓ | 本周 ≥ 3次 + 近10条 ≥ 4条 + 均时 > 25min + 异常率 ≤ 50% + α ≥ 0.55 + **综合榜 Top 10 自动退出** |

**关键概念：**
- **α（基线可信度）**：`有效记录数 / (有效记录数 + 8)`，记录越多越稳定，最大趋近 1.0
- **异常练琴**：单次练琴 `is_outlier = true`，包括 `too_long`（时长异常超长）和 `meal_break`（含饭点时段）
- `RANK()` 函数：完全相同的多维排序值才会真正并列，加入多级排序后并列概率极低
- **综合榜 Top 10 退出专项榜**：综合榜排名前 **10** 的学生自动从进步榜、稳定榜、守则榜中排除（`comp_top10` CTE 实现，FIX-65）。设计原则：综合榜前十名已获最高荣誉，三个专项榜名额留给其他有特定优势的学生，让更多学生有机会被表彰。
- **进步榜排序键**：按**绝对涨分**（`display_score - lw_composite`）降序，而非百分比涨幅。防止上周几乎不练的学生以极低基数制造虚高百分比霸榜。`trend_score` 列仍存储百分比涨幅（`(本周-上周)/上周×100`，保留1位小数）供前端展示 `+XX.X%`，但不参与排名。
- **进步榜基准**：取本周之前最近 2 个有效周的综合分 **MAX** 作为对比基准（见 `last_week_scores`），配合绝对涨分排序抑制低基数虚高。
- **假期/低分周防虚高处理**：进步榜基准取本周之前**最近 2 个有效周的最高综合分（MAX）**，而非最近 1 周。这样节后第一周偶发低分不会成为基准，必须真正超越近期最佳才能上进步榜；同时最多回溯 12 周，寒暑假后首周不会因无历史数据导致进步榜空白。

## 音符币系统

### 音符币涉及数据库表

#### 1. `student_coins`（学生余额表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `student_name` | TEXT PK | 学生姓名（主键） |
| `balance` | INTEGER | 当前音符币总余额，默认 0，**可跨学期积累** |
| `semester_earned` | INTEGER | 本学期通过排行榜/管理员正向调整累计获得的音符币，默认 0，**学期末由管理员重置** |
| `updated_at` | TIMESTAMPTZ | 最后更新时间 |

**特点：**
- 每个学生只有一行记录
- `balance` 永远是最新总余额，跨学期有效，可随时兑换奖励
- `semester_earned` 仅统计正向加法（正数 amount），兑换扣除不影响此值
- 梅纽因之星以 `semester_earned` 为唯一判断依据，而非 `balance`

#### 2. `coin_transactions`（音符币流水记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID PK | 主键（自动生成） |
| `student_name` | TEXT | 学生姓名 |
| `amount` | INTEGER | 变动数量（**正数=增加，负数=扣除**） |
| `balance_after` | INTEGER | 变动后的最新余额（冗余存储，方便展示无需回溯计算） |
| `reason` | TEXT | 备注说明（必填，如"综合榜第一名奖励"） |
| `transaction_type` | TEXT | `'manual'`=管理员手动操作，`'auto_reward'`=系统自动结算 |
| `created_at` | TIMESTAMPTZ | 记录创建时间 |

**索引：**
- `idx_coin_tx_student`：`(student_name, created_at DESC)` — 按学生查流水，时间倒序

**RLS（行级安全策略）：**
- 读取：允许所有人（`anon` + `authenticated`）查询
- 写入：仅通过 `adjust_student_coins` RPC 函数写入（SECURITY DEFINER 权限）

### 音符币涉及数据库函数与视图

#### `public.adjust_student_coins(p_student_name, p_amount, p_reason, p_type)`

**作用：** 原子操作，确保"更新余额"和"写入流水"同时成功，不会出现数据不一致。

**文件：** `setup_coins_db.sql`

**参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `p_student_name` | TEXT | 学生姓名 |
| `p_amount` | INTEGER | 变动数量（正=加，负=减） |
| `p_reason` | TEXT | 备注说明 |
| `p_type` | TEXT | 操作类型，默认 `'manual'`；自动结算传 `'auto_reward'` |

**返回：** `INTEGER`（变动后的最新余额）

**内部执行步骤：**
1. `INSERT ... ON CONFLICT DO NOTHING`：若学生不在余额表，先初始化为 0
2. `SELECT ... FOR UPDATE`：行锁，防止并发修改
3. 计算新余额：`v_new_balance := v_current_balance + p_amount`
4. `UPDATE student_coins`：更新余额 + **当 `p_amount > 0` 时同步累加 `semester_earned`**（兑换扣除不影响学期累计）
5. `INSERT coin_transactions`：写入流水记录
6. 返回新余额

> **注意**：`adjust_student_coins` 最新定义在 `setup_semester.sql` 中（`CREATE OR REPLACE`），
> 同时对 auto_reward 和 manual 两种正向调整均累加 `semester_earned`，以支持管理员手动补发。

**权限：** `SECURITY DEFINER` + `GRANT EXECUTE TO anon, authenticated`

#### `public.vw_student_coin_balances`（视图）

**作用：** 将 `student_database`（学生基础信息）与 `student_coins`（余额）联合查询，方便管理后台一次性加载学生列表。

```sql
SELECT
    sd.name                         AS student_name,
    sd.major                        AS student_major,
    sd.grade                        AS student_grade,
    COALESCE(sc.balance, 0)         AS balance,           -- 总余额，跨学期
    COALESCE(sc.semester_earned, 0) AS semester_earned,   -- 本学期已获得
    sc.updated_at
FROM student_database sd
LEFT JOIN student_coins sc ON sd.name = sc.student_name;
```

**权限：** `GRANT SELECT TO anon, authenticated`
> Supabase 后台会显示 `UNRESTRICTED` 红色标签，属于正常现象，不影响功能。

#### `public.award_meiyin_star(p_semester, p_note)`

**作用：** 学期末颁发梅纽因之星荣誉，自动找出本学期 `semester_earned` 最高且 ≥ 400 的学生并写入记录。

**文件：** `setup_semester.sql`

**参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `p_semester` | TEXT | 学期标识，如 `"2026春"` |
| `p_note` | TEXT | 颁奖备注，可为 NULL |

**返回：** TEXT — 颁发结果说明（含获奖学生姓名与学期累计币数）

**执行逻辑：**
1. 检查 `meiyin_star_log` 中该学期是否已颁发（UNIQUE 约束防重复）
2. 找 `semester_earned ≥ 400` 中最高者（同分按 `updated_at ASC` 先到先得）
3. 若无人达标，返回提示并附当前前三名供参考
4. 写入 `meiyin_star_log` 记录

**权限：** `SECURITY DEFINER` + `GRANT EXECUTE TO anon, authenticated`

#### `public.start_new_semester(p_confirm)`

**返回：** TEXT — 重置结果（含重置人数）

**注意：** 此操作不可撤销，执行前务必确认本学期梅纽因之星已颁发完毕。

### 音符币自动结算

#### `public.weekly_coin_reward_log`（结算防重复记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID PK | 主键 |
| `week_monday` | DATE UNIQUE | 该周周一，**唯一键**防重复 |
| `rewarded_at` | TIMESTAMPTZ | 实际结算时间 |
| `total_events` | INTEGER | 本周发放总次数（同学生多榜算多次） |
| `total_coins` | INTEGER | 本周发放总音符币数 |
| `summary` | JSONB | 各榜人次/金额，格式：`{"综合榜":{"人次":8,"币":480},...}` |

#### `public.reward_weekly_coins()` 结算函数

**文件：** `setup_coin_rewards.sql`

**触发方式：** pg_cron 定时 + 支持管理员手动调用

**执行逻辑：**
1. 读取北京时间本周一/五日期
2. **开始日期保护**：`v_friday_bjt < 2026-04-03` 时直接返回，不执行
3. **防重复**：若 `weekly_coin_reward_log` 已有本周记录则跳过
4. 调用 `get_weekly_leaderboards()` 获取四榜实时数据
5. 遍历每条上榜记录，按名次规则确定金额和流水说明
6. 调用 `adjust_student_coins(p_type='auto_reward')` 原子写入
7. 在 `weekly_coin_reward_log` 插入本周结算记录
8. 返回文字摘要（各榜人次/总金额）

**发放规则：**

| 名次 | 综合榜 | 稳定榜 | 守则榜 | 进步榜 |
|------|--------|--------|--------|--------|
| 第 1 名 | 100 币（榜首霸主） | 50 币 | 45 币 | 40 币 |
| 第 2–3 名 | 80 币（荣耀亚军） | 35 币 | 30 币 | 25 币 |
| 第 4–6 名 | 60 币（实力季军） | 20 币 | 18 币 | 15 币 |
| 第 7–10 名 | 40 币（优秀达人） | — | — | — |

**流水说明（reason 字段）示例：**
- 综合榜：`【周榜结算】2026年04月03日当周 · 综合榜第1名（榜首霸主）· 综合分 85.5 分`
- 稳定榜：`【周榜结算】2026年04月03日当周 · 稳定榜第2名 · α 可信度 0.893`
- 守则榜：`【周榜结算】2026年04月03日当周 · 守则榜第3名 · 近10次异常率 0.0%`
- 进步榜：`【周榜结算】2026年04月03日当周 · 进步榜第1名 · 本周进步 +24 分`

## 定时任务

| 任务名称 | Cron 表达式（UTC） | 北京时间 | 执行函数 | 说明 |
|---------|-----------------|---------|---------|------|
| `backup_weekly_leaderboards_job` | `30 13 * * 5` | 每周五 21:30 | `backup_weekly_leaderboards()` | 将当周四榜快照写入 `weekly_leaderboard_history` |
| `reward_weekly_coins_job` | `32 13 * * 5` | 每周五 21:32 | `reward_weekly_coins()` | 按名次发放音符币；正式开始日期 2026-04-03 |

> 两个任务相差 2 分钟，`reward_weekly_coins()` 内部直接调用 `get_weekly_leaderboards()` 取实时数据，与备份任务完全独立，先后顺序不影响结果。

**管理命令：**
```sql
-- 查看所有定时任务
SELECT jobid, jobname, schedule, command, active FROM cron.job;

-- 手动触发备份（测试用）
SELECT public.backup_weekly_leaderboards();

-- 手动触发音符币结算（测试/补发）
SELECT public.reward_weekly_coins();

-- 查看最近结算日志（pg_cron >= 1.4）
SELECT * FROM cron.job_run_details
WHERE jobid = (SELECT jobid FROM cron.job WHERE jobname = 'reward_weekly_coins_job')
ORDER BY start_time DESC LIMIT 10;

-- 查看历次结算记录
SELECT week_monday, rewarded_at, total_events, total_coins, summary
FROM public.weekly_coin_reward_log ORDER BY week_monday DESC;

-- 删除某周记录（允许重新结算，谨慎操作）
-- DELETE FROM public.weekly_coin_reward_log WHERE week_monday = '2026-03-30';

-- 取消任务
SELECT cron.unschedule('backup_weekly_leaderboards_job');
SELECT cron.unschedule('reward_weekly_coins_job');
```
